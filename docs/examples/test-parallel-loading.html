ToolBox API Test - Parallel Execution & Loading Screen

1. Loading Screen Test

Test the showLoading and hideLoading functions. The loading screen should appear with a spinner and custom message, then disappear after 3 seconds.

// Show loading await toolboxAPI.utils.showLoading('Processing...'); // Simulate work await delay(3000); // Hide loading await toolboxAPI.utils.hideLoading();

2. Parallel Execution Test (Mock)

Test the executeParallel function with mock operations. In a real tool, you would use actual API methods like dataverse.retrieve or connections.getActiveConnection.

// Execute multiple operations in parallel const [theme, connection] = await toolboxAPI.utils.executeParallel( toolboxAPI.utils.getCurrentTheme(), toolboxAPI.connections.getActiveConnection() );

3. Combined Test

Test both features together: show a loading screen while executing multiple operations in parallel. This is the recommended pattern for fetching data in tools.

// Show loading await toolboxAPI.utils.showLoading('Fetching data...'); try { // Execute operations in parallel const [theme, connection] = await toolboxAPI.utils.executeParallel( toolboxAPI.utils.getCurrentTheme(), toolboxAPI.connections.getActiveConnection() ); // Process results... } finally { // Always hide loading await toolboxAPI.utils.hideLoading(); }

4. Error Handling Test

Test error handling with try-finally to ensure loading screen is hidden even when errors occur.

await toolboxAPI.utils.showLoading('Testing error...'); try { // Simulate error throw new Error('Test error'); } catch (error) { console.error(error); } finally { // Loading screen should still be hidden await toolboxAPI.utils.hideLoading(); }
